Research and Action

I found a lot of useful techniques online to help me with my first technical exercise. The first thing that helped me was researching information on media queries. I then used this information to change certain attributes at certain breakpoints. I researched what sizes typical device were eg. tablet 768px , mobile 480px. Then used this to change the background colour at different points. For my website, at full sized desktop, the colour is a pale orange whilst the text is black. Was the viewport is decreased the size background colour changes to blue and the text changed to white.

I used information from stack-overflow to find out how to fetch the live viewport pixels. I tried this with native javascript first but this only seemed to work when refreshing the page. I then found another solution using jQuery, which seemed to work perfectly.


Some useful websites that helped me along the way:

Building a Basic Responsive Website

Displaying the viewport using jQuery - Realtime
This helped me display the viewport in realtime whilst resizing the screen. I orginally tried it with native JS but jQuery seemed to work better.

Making the site display properly on mobile I couldn't figure out why my site wasnt showing the attributes of my mobile queries. I did a quick google and found that this meta tag needed to be used meta name="viewport" content="width=device-width, initial-scale=1.0"> which seemed to fix the issue.

The thing that I learnt the most from this exercise was actually using media queries and understanding how they work with different screen dimensions. By placing each element/class inside each query it would then apply it as the window was resized.
